# GitHub Discussions Categories

Use the following categories to keep support and product conversations organized.

## 1) Announcements
- Purpose: Releases, roadmap updates, breaking changes.
- Maintainers only for creating topics.

## 2) Q&A
- Purpose: User questions, setup help, usage clarifications.
- Target response: first maintainer response within 24 business hours.

## 3) Ideas
- Purpose: Feature proposals and workflow improvements.
- Include: problem statement, proposed behavior, expected impact.

## 4) Show and Tell
- Purpose: Community examples, integrations, and success stories.

## 5) Troubleshooting
- Purpose: Diagnostic help for runtime/config issues.
- Required: command used, sanitized config, full error output.

## 6) Integrations (CI/CD, Docker, Cloud)
- Purpose: Pipeline templates, containerization, and deployment patterns.

## Moderation Rules
- Be respectful and constructive.
- No secrets or private keys in posts.
- Close resolved troubleshooting threads after confirmation.
